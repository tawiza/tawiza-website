C'est un projet open source. Pas une startup.

Personne ne lève de fonds. Personne ne prépare un pivot. On construit un outil d'intelligence territoriale parce que les données publiques françaises méritent mieux qu'un portail open data que personne ne visite.

On cherche des gens, pas des profils LinkedIn

Dev Python

Le backend est en FastAPI + Ollama. Chaque API gouvernementale est un connecteur indépendant. Si tu as déjà bataillé avec une API publique française et ses formats improbables, tu sais exactement de quoi on parle.

Dev frontend

Le site est du HTML/CSS/JS statique. Les visualisations sont en D3.js/SVG. Si tu veux faire des graphiques qui racontent une histoire, c'est ici. Pas de framework, pas de build de 45 secondes. Tu ouvres le fichier, tu modifies, tu recharges.

Voir les issues frontend

Data scientist

Les pipelines de données sont le coeur du projet. Scraping, normalisation, croisement. Si tu sais transformer un CSV illisible en quelque chose d'utile, on a du travail. Et si tu as déjà essayé de croiser des données INSEE avec des données BODACC, tu sais que "du travail" est un euphémisme.

Tu n'es pas dev ?

On a aussi besoin de gens qui écrivent bien, qui vérifient les sources, qui connaissent leur territoire. Une analyse économique locale écrite par quelqu'un qui vit sur place vaut dix dashboards générés automatiquement.

Voilà comment contribuer sans écrire une ligne de code :

  • Signaler une erreur dans une analyse → ouvrir une issue sur GitHub
  • Proposer un sujet d'analyse → GitHub Discussions
  • Relire et vérifier les sources d'un article
  • Écrire une analyse sur ton territoire → nous contacter avec tes données et ton angle

Comment commencer

1. Cloner le dépôt

Le code est sur GitHub. Un git clone, un docker compose up, et c'est parti. Le README explique le reste.

2. Choisir une issue

Les issues tagguées good first issue sont faites pour ça. Elles sont documentées, délimitées, et personne ne va te juger si tu poses des questions.

3. Proposer une PR

Pas de process bureaucratique. Tu fais ta branche, tu pousses, tu ouvres une PR. On relit, on discute, on merge. C'est aussi simple que ça.

Pour l'instant, Tawiza est un projet solo. Chaque contribution, même minime, fait une vraie différence. Si tu contribues, tu seras parmi les premiers.
